Quiero mejorar la lógica del sistema de PlanDocente IA, no solo la interfaz. Necesito que funcione como un sistema completo donde todo lo que se genera se guarda, se puede editar y se puede reutilizar.

Hacer estos cambios:

1) Reordenar el menú superior con este orden exacto:
- Escuelas y Cursos
- Alumnos
- Planificaciones
- Registro de clase
- Informes

2) En la sección Informes:
- mantener la generación automática del informe a partir de registros de clase
- permitir editar manualmente el informe generado en un área de texto editable
- agregar botón “Guardar informe”
- agregar botón “Descargar PDF”
- agregar botón “Volver a editar”
- cuando se guarde, que el informe quede almacenado en la app

3) Crear un sistema de historial general dentro de la app:

Cada módulo debe tener su propio historial:
- Historial de planificaciones
- Historial de registros de clase
- Historial de informes

Y además crear una vista global llamada:
“Actividad reciente” o “Historial general”

4) El historial debe guardar como mínimo:
- id único
- fecha de creación
- fecha de modificación
- escuela
- curso
- alumno (si corresponde)
- tipo (planificación / registro / informe)
- título o resumen
- contenido completo
- estado (borrador o final)

5) En Planificaciones:
- permitir guardar cada planificación
- mostrar listado de planificaciones anteriores por curso
- permitir volver a abrir una planificación y editarla
- permitir duplicar una planificación existente

6) En Registro de clase:
- permitir guardar cada registro
- mostrar historial por curso y fecha
- permitir editar registros previos
- opcional: vincular registro con una planificación existente

7) En Informes:
- mostrar historial de informes guardados por alumno
- permitir abrir un informe anterior
- permitir editarlo nuevamente
- permitir duplicarlo
- descargar cualquier informe guardado en PDF

8) Crear una sección nueva: “Actividad reciente” o “Historial general”
- mostrar últimos elementos creados (planificaciones, registros, informes)
- ordenar por fecha descendente
- mostrar tipo de contenido (icono o etiqueta)
- permitir abrir directamente cada elemento
- permitir filtrar por:
  - curso
  - alumno
  - tipo de contenido

9) Agregar persistencia real:
- no perder la información al cambiar de sección
- todo lo guardado debe permanecer disponible al recargar la app
- usar almacenamiento persistente del proyecto (backend o base de datos)
- si no existe backend, crear una estructura simple (ej: local storage o base interna)

10) UX y funcionamiento:
- botones claros: guardar, editar, duplicar, descargar
- mostrar mensaje de “guardado exitoso”
- mostrar estado vacío si no hay datos
- diseño simple, limpio y profesional
- no romper el funcionamiento actual

11) IMPORTANTE:
- no solo mostrar contenido generado en pantalla
- cada planificación, registro o informe debe poder:
  - guardarse
  - editarse
  - volver a abrirse
  - reutilizarse

La app debe pasar de ser un generador de contenido a un sistema completo de gestión docente.
